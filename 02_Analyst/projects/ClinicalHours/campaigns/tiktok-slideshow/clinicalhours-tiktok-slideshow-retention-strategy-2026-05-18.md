# ClinicalHours TikTok Slideshow Retention Strategy - 2026-05-18

## Goal

Adapt the best parts of the Felt slideshow psychology into a lane-aware ClinicalHours system that increases swipe-through without weakening product truth.

## Interactive User Intake

Before building slides for Shivam directly, ask what content or topic the deck should cover.

If that answer still leaves the assignment ambiguous, ask one short follow-up about:

- the audience
- the CTA

Only fall back to the default lane when the work is automated or the user is unavailable.

## Findings First

What transfers from Felt:

- fast problem recognition
- fast payoff
- visible proof progression
- screenshot-first copy discipline
- calm CTA instead of overhyped pressure

What changes for ClinicalHours:

- the "reward" is clarity and control, not excitement
- the buyer loop is operational relief, not entertainment return
- trust and specificity matter more than cleverness

## Core Sequence Logic

### For `clinic_ops_primary`

Use:

1. friction
2. unification
3. workflow proof
4. control
5. activation

Viewer questions:

1. why is this still so manual?
2. what does the cleaner system look like?
3. does it actually handle the workflow?
4. can I trust it operationally?
5. is there a believable next step?

### For `student_discovery_secondary`

Use:

1. friction
2. discovery
3. simplification
4. documentation direction
5. activation

Viewer questions:

1. why is finding clinical experience still this scattered?
2. where does ClinicalHours make the search easier?
3. does it actually simplify the first steps?
4. does it connect to the rest of the process?
5. what should I do next?

## What Makes The Swipe Happen

### 1. Answerable tension

Good ClinicalHours hooks:

- "Clinic ops still run by hand."
- "Finding clinical hours is still manual."
- "Software was built for everyone except free clinics."

Bad hooks:

- vague mission statements
- giant market-size claims with no immediate human pain

### 2. Immediate structural payoff

Slide 2 should not just restate the pain.

It should show:

- one platform
- one map
- one queue
- one operating layer

The viewer should feel the deck is getting more useful, not more abstract.

### 3. Deeper proof, not louder proof

As the deck progresses:

- the screenshots should get more specific
- the claims should get narrower and more believable
- the product should feel more real

### 4. Calm closure

ClinicalHours should end with:

- request demo
- see the platform
- browse as guest
- get started

Not with:

- fake urgency
- investor hype
- medical-school anxiety bait

## Copy Rules

- Headline should usually be `4-8` words.
- Subtext should usually be one short sentence.
- One promise per slide.
- Structural language beats clever language.

Preferred clinic-ops language:

- coordinator
- workflow
- onboarding
- credentialing
- status
- audit trail
- queue
- compliance
- team

Preferred student language:

- find
- map
- opportunities
- apply
- track
- start
- browse
- clinical future

## Visual Rules

- One hero proof asset per slide.
- Maximum two support proof crops.
- The hero proof must do most of the persuasion.
- Support crops should add evidence, not decoration.
- Generated support imagery can strengthen rhythm or atmosphere, but it must stay secondary to truthful captured proof.
- Text should never depend on tiny interface detail to land.
- Slide rhythm should vary enough to keep attention.

## Strong Lane Defaults

### `clinic_ops_primary`

- dark stage
- blue + green accents
- larger desktop proof
- dense proof only when still readable
- CTA framed as `request a demo`, not `buy now`

This is only the legacy default.

If Shivam explicitly wants a brighter direction, or if the task points to `clinicalhours_visual_strategy_2026-05-19.md`, override the dark-stage default and use a brighter editorial healthcare palette while preserving clinic-ops credibility.

### `student_discovery_secondary`

- cleaner hero surfaces
- map and auth surfaces
- more open space
- less operational jargon
- CTA framed as `get started` or `browse`

## Anti-Patterns

- mixing clinic and student copy on the same slide
- using placeholder metrics as proof
- implying HIPAA certification that is not visibly supported
- implying guaranteed admissions, placements, or clinical hours
- fake dashboard redraws
- fake partner logos
- copy that outruns the visible asset
