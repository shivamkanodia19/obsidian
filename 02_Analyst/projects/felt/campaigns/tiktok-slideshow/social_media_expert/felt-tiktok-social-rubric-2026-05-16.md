# Felt TikTok Social Rubric - 2026-05-16

## Purpose

This rubric turns the existing Felt strategy and TikTok slideshow research into a grading system that agents can use to:

- score one candidate slide at a time
- group passed slides into one valid five-slide deck
- reject weak or risky work with specific reasons
- release only deck-level winners for email

This is intentionally **product-first**, **compliance-aware**, and **truthful to the real Felt UI**.

## Non-Negotiable Principles

- Felt must read as a `private tables + training + decision-practice` product, not a gambling promotion.
- Slides must use real Felt UI states. No fabricated flows, fake proof, or broken states.
- Copy must stay away from money, winnings, payout energy, or casino-coded hype.
- Screenshots must do real explanatory work. Decorative proof does not count as proof.
- Each slide must be understandable in about `2 seconds`.
- The deck should feel premium, controlled, and credible, not loud, tacky, or artificially addictive.

## Required Five-Slide Order

Every approved deck must include exactly one slide in this order:

1. `friction`
2. `simplification`
3. `operational_proof`
4. `mastery`
5. `return`

## Role Definitions

### 1. Friction

Job:

- name the mess
- create recognition
- create a clean curiosity gap

Best proof:

- home or organization state that suggests order is possible

Avoid:

- generic category claims
- busy table chaos as the hero proof

### 2. Simplification

Job:

- resolve the tension fast
- prove setup is light and understandable

Best proof:

- code, join, host, rejoin, or one-screen lobby clarity

Avoid:

- copy that promises ease without visible setup proof

### 3. Operational Proof

Job:

- prove the product stays legible in actual use
- show table state, room structure, or action clarity

Best proof:

- visible turn state
- visible stacks or blinds
- room state that feels organized

Avoid:

- tiny crops that hide the actual use state

### 4. Mastery

Job:

- show Felt is not only logistics
- show that users get sharper through training or review

Best proof:

- trainer UI
- feedback UI
- session review or coaching surfaces

Avoid:

- vague "play smarter" claims with weak proof

### 5. Return

Job:

- make return feel social, credible, and natural
- close with invitation instead of pressure

Best proof:

- clean home or return surface
- private table continuity
- practice adjacency that gives the group a reason to come back

Avoid:

- salesy CTA energy
- urgency without product reason

## Slide Scoring Scale

Use a `1-5` scale for each category:

- `1` = fail
- `2` = weak
- `3` = needs revision
- `4` = pass
- `5` = strong pass

## Slide Rubric

### A. Compliance

Weight: `30%`

Pass signals:

- product-led framing
- no real-money or profit framing
- no casino tropes
- truthful UI state
- age-safe, non-exploitative tone

Automatic rejection examples:

- `win money`
- `cash out`
- `beat the casino`
- fake or fabricated UI
- broken/error state used as hero proof

### B. Message Clarity

Weight: `20%`

Pass signals:

- one dominant idea
- headline carries the slide quickly
- meaning is obvious in about `2 seconds`
- subtext supports instead of rescuing the slide

Weak signals:

- crowded text
- two competing messages
- nice typography but unclear takeaway

### C. Product-Led Proof

Weight: `25%`

Pass signals:

- screenshot clearly proves the claim
- hero proof asset is legible
- support crops add real explanatory value
- proof is stronger than styling

Weak signals:

- screenshot feels decorative
- crop is technically real but not persuasive
- headline says one thing while proof shows something else

### D. Visual Restraint

Weight: `15%`

Pass signals:

- premium dark stage
- controlled contrast
- sparse gold accents
- felt green used for continuity and competence
- strong eye path from headline to proof

Weak signals:

- noisy glow
- tacky visual effects
- casino-neon energy
- repeated layout rhythm with no fresh emphasis

### E. Technical Export

Weight: `10%`

Pass signals:

- `1080 x 1920`
- sharp text
- no clipping
- safe-zone compliance
- no text overflow
- no text/proof overlap
- no browser chrome or export artifacts

Automatic rejection examples:

- wrong dimensions
- clipped copy
- visible broken UI
- essential text outside safe area

## Slide Decision Rules

### Approve

Approve the slide only if:

- all hard technical and truthfulness checks pass
- weighted score is at least `4.2`
- every category score is at least `4`
- the slide actually matches its intended role

### Revise

Revise when:

- hard fail rules do not trigger
- weighted score is between `3.0` and `4.19`, or
- one category is `3` while the rest are healthy

### Reject

Reject when:

- any hard fail rule triggers, or
- weighted score is below `3.0`, or
- role mismatch is severe enough that the slide does not belong in its assigned slot

## Deck Assembly Rules

- A deck must contain exactly `5` slides.
- A deck must use the required order: `friction -> simplification -> operational_proof -> mastery -> return`.
- Use only slides that already passed slide-level approval.
- Prefer one hero proof asset per slide.
- Maximum `2` support proof crops per slide.
- Do not repeat the same hero state in adjacent slides.
- Do not let two adjacent slides solve the same emotional job.
- The final slide may contain the strongest CTA, but it still has to feel credible and product-led.

## Deck Rubric

### A. Role Coverage And Order

Weight: `20%`

Pass signals:

- all five required roles are present
- role order is correct
- each slide belongs in its assigned slot

### B. Narrative Flow

Weight: `25%`

Pass signals:

- slide `1` creates curiosity
- slide `2` pays it off quickly
- slide `3` proves control
- slide `4` proves improvement
- slide `5` gives a believable reason to return

### C. Visual Variety

Weight: `15%`

Pass signals:

- the deck feels like one system
- slides are not near-duplicates
- layout rhythm changes enough to preserve attention

### D. Proof Progression

Weight: `20%`

Pass signals:

- proof gets more specific as the deck progresses
- the viewer feels they are going deeper into the product
- no slide exists only for decoration

### E. Return And CTA Credibility

Weight: `20%`

Pass signals:

- final slide closes with calm invitation
- group return logic is visible
- CTA does not sound like paid-ad desperation

## Deck Decision Rules

Approve the deck only if:

- all five slides are already slide-approved
- all five required roles are present in order
- weighted deck score is at least `4.3`
- every deck category score is at least `4`

Revise the deck when:

- the slides are individually good but sequencing, redundancy, or handoff quality weakens the set

Reject the deck when:

- role coverage fails
- order fails
- the narrative stalls
- the set feels repetitive, casino-coded, or weakly proven as a whole

## Email Release Rule

Email only when:

- one full deck is deck-approved
- all five slides in that deck are slide-approved
- the deck has not already been emailed

Do not email:

- single slides
- partially passed decks
- decks with unresolved revision notes

## What Agents Should Store In JSON

At minimum, each slide or deck grade record should capture:

- `asset_id` or `deck_id`
- intended role
- source captures
- copy used
- hard-gate results
- category scores
- weighted score
- status: `approved`, `revise`, or `rejected`
- rejection reasons or revision notes
- email state for approved decks
