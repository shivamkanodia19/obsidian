# ClinicalHours Summer 2026 Premium Problem-First Strategy Review

## Why this direction changed

The earlier ClinicalHours slideshow direction was still too close to a product explainer:

- it introduced the interface too early
- it asked the viewer to process too much information before they felt the problem
- it looked designed, but not yet like a premium campaign

This revised direction shifts the first attention battle away from explanation and toward recognition:

- `I need clinical hours`
- `the search feels scattered`
- `good openings move before I do`

Only after those beats land does the product appear.

## What the Scratch AI research changed

The main lesson from Scratch AI Golf's public marketing is not "use AI art."

It is:

- sell relief from a stressful decision first
- build a premium visual world around that feeling
- keep each frame focused on one promise
- let the mechanism show up after the emotional wedge

Scratch's public marketing consistently feels like `problem pressure -> simple promise -> polished proof`, not `feature dump -> explanation -> CTA`.

That is the closest transferable lesson for ClinicalHours.

## New slideshow structure

This build uses a `3 + 3` sequence:

1. emotional hook
2. access friction
3. time pressure
4. map reveal
5. real opportunity proof
6. low-pressure CTA

This is intentionally different from an informational carousel. The first half is there to make the student feel seen. The second half is there to prove that ClinicalHours reduces the friction.

## Creative choices

### 1. Relevant aesthetic images first

The opening slides use AI-generated editorial visuals of:

- a pre-health student searching late at night
- a student outside a clinical threshold
- a semester-pressure still life

These images make the problem feel immediate without explaining the product too soon.

### 2. One promise per frame

The copy stays short and singular:

- `Everyone says get clinical hours.`
- `Still outside looking in.`
- `The good openings move first.`
- `Start with what's actually nearby.`
- `See what's real before you apply.`
- `Browse first. Save the good ones.`

This follows the same high-signal framing that made Scratch's public assets feel strong.

### 3. Screenshots arrive late

ClinicalHours screenshots do not appear until slide `4`.

That matters because it lets the user first understand the problem emotionally, then experience the product as relief instead of homework.

### 4. Darker premium palette

The new deck uses a darker, more premium system:

- deep evergreen-black background
- warm parchment/cream typography
- sand accent rule
- restrained mint proof accents

This is closer to premium app-store marketing than to a bright informational student deck.

## Areas still worth improving

- The generated lifestyle scenes are strong, but a future pass could improve continuity by using one recurring character or campaign world across all three opening slides.
- The final CTA slide works well for awareness, but a paid-social test might still want a tighter `5`-slide version.
- If ClinicalHours has stronger live proof states available later, slide `5` could become even stronger with richer opportunity detail or verified student feedback.

## Plugin recommendations

### 1. `Figma`

Best use:

- fast exploration of multiple higher-end visual routes
- editable Figma Slides versions for collaboration
- testing different premium campaign languages without rebuilding code each time

Used in this pass:

- generated editable deck options in Figma Slides using the `Figma` deck generator

### 2. `Browser`

Best use:

- refreshing public ClinicalHours screenshots
- checking live visual states before new exports

### 3. `Chrome`

Best use:

- authenticated capture states
- repeatable proof gathering when logged-in views matter

### 4. `Manus`

Best use:

- fast alternate deck concepts
- rapid exploration of adjacent campaign directions

This is optional, but useful if you want many style variants quickly.
