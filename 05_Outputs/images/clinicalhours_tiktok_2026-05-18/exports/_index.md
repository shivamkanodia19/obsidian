# ClinicalHours TikTok Exports

Versioned slide exports for the ClinicalHours TikTok slideshow.

## Versions

- `v1/` - canonical `clinic_ops_primary` export set with the strongest documentation coverage
- `v2auth/` - authenticated student-lane export set documented in `clinicalhours_tiktok_iteration_notes_v2auth.md`
- `v4onboard/` - light-only onboarding-visibility `clinic_ops_primary` export set documented in `clinicalhours_tiktok_iteration_notes_v4onboard.md`
- `v4expiry/` - light-only expiring-clearances `clinic_ops_primary` export set documented in `clinicalhours_tiktok_iteration_notes_v4expiry.md`
- `v4impact/` - issue-specific student-facing export set for impact-story capture and later recall, documented in `clinicalhours_tiktok_iteration_notes_v4impact.md`
- `v4memory/` - bright student search-memory export set documented in `clinicalhours_tiktok_iteration_notes_v4memory.md`
- `v5firstpath/` - student first-path export set documented in `clinicalhours_tiktok_iteration_notes_v5firstpath.md`
- `v6realleads/` - student trust-focused export set documented in `clinicalhours_tiktok_iteration_notes_v6realleads.md`
- `v6guestfirst/` - student guest-first export set documented in `clinicalhours_tiktok_iteration_notes_v6guestfirst.md`
- `v7semesterstart/` - student semester-start export set documented in `clinicalhours_tiktok_iteration_notes_v7semesterstart.md`
- `v8conciseops/` - concise light clinic-ops experiment documented in `clinicalhours_tiktok_iteration_notes_v8conciseops.md`; keep as a cautionary branch, not the preferred baseline
- `v9impactops/` - impact-led concise clinic-ops correction documented in `clinicalhours_tiktok_iteration_notes_v9impactops.md`
- `v10fallimpact/` - high-impact student fall discovery experiment documented in `clinicalhours_tiktok_iteration_notes_v10fallimpact.md`
- `v3bright/` through `v3bright7/` - bright student-facing iteration history aligned with `docs/playbooks/clinicalhours_visual_strategy_2026-05-19.md`

## Latest Set

- `v3bright7` is the current bright canonical export directory and is backed by `clinicalhours_tiktok_iteration_notes_v3bright7.md` plus a machine-readable deck spec.
- `v1` remains the safest starting point when a future agent needs the most reproducible deck path.
- `v4onboard` is the issue-specific clinic-ops branch for onboarding visibility and preserves the light-only direction requested on `2026-05-20`.
- `v4expiry` is the issue-specific clinic-ops branch for expiring clearances and keeps the same light-only direction while centering renewal visibility.
- `v4impact` is the issue-specific student branch for students who need to log hours, reflections, and account details before they later explain impact.
- `v4memory` is the issue-specific bright student branch for losing track after discovery and keeps the search-memory story tighter than `v3bright7`.
- `v5firstpath` is the issue-specific student branch for the first practical path from first click through first logged shift; its `2026-05-30` refresh tightened slide `3` and slide `4` around more specific proof.
- `v6realleads` is the issue-specific student trust branch for stale lists, lead quality, and proving a clinic is worth the click, but the `2026-05-30` audit recommends revising or retiring it before reuse because its trust claim currently outruns the proof.
- `v6guestfirst` is the issue-specific student activation branch for browsing before signup and reframing account creation as optional follow-through.
- `v7semesterstart` is the issue-specific student semester-start branch for students looking for hours during the semester; it passed a fresh judge loop on `2026-06-01` after rebuilding slide `3` and slide `4` around one Houston Methodist proof chain.
- `v8conciseops` is a documented concise clinic-ops experiment, but it is not the preferred future baseline because the slides ended up too visually similar and the screenshot text still reads too slowly.
- `v9impactops` is the current concise clinic-ops candidate. It was rebuilt on `2026-06-02` to keep the shorter-copy goal while using clearer queue, review, output, and trust proof than `v8conciseops`.
- `v10fallimpact` is the newest student high-impact aesthetic branch. It passed render QA on `2026-06-07`, but it should get a fresh-context judge loop before it replaces `v7semesterstart` as the safest student baseline.

## Quality Baseline

- Use `v7semesterstart` as the quality benchmark when a future agent needs a deck that feels more impactful, more varied, and easier to parse slide to slide.
- `v9impactops` is the current clinic-ops example of that rule being applied: fewer words than `v7semesterstart`, but more visual variety and clearer proof than `v8conciseops`.
- `v10fallimpact` is the current student example of how to modernize the shell without changing the trusted proof sequence; treat it as the preferred aesthetic experiment, not the default student canonical branch yet.
- For future concise asks, the target is `v7semesterstart`-level proof clarity and pacing with fewer words than `v7semesterstart`, not `v8conciseops`-style shell repetition.
